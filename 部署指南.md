# Google Ads 连接器部署指南

## 版本信息

- **版本**: v1.0
- **描述**: 推送客户匹配数据到 Google Ads 平台，支持 SHA-256 加密邮箱、手机号、地址及移动设备 ID
- **分类**: Marketing

## 环境要求

### 1. 基础环境

- **Java**: JDK 17
- **Maven**: 3.8+
- **PostgreSQL**: 15+
- **Nacos**: 2.5.1+
- **Docker**: 20.10+ (可选)
- **Docker Compose**: 2.0+ (可选)

### 2. 硬件要求

- **CPU**: 2 核心以上
- **内存**: 4GB 以上
- **磁盘**: 20GB 以上

## 部署步骤

### 1. 安装 PostgreSQL 数据库

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install postgresql-15

# CentOS/RHEL
sudo yum install postgresql15-server
sudo postgresql-setup initdb
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

### 2. 创建数据库和用户

```bash
# 切换到 postgres 用户
sudo -u postgres psql

# 创建数据库
CREATE DATABASE livepulse_google_ads;

# 创建用户
CREATE USER google_ads_user WITH PASSWORD 'your_password';

# 授权
GRANT ALL PRIVILEGES ON DATABASE livepulse_google_ads TO google_ads_user;

# 退出
\q
```

### 3. 执行数据库脚本

```bash
# 进入 SQL 脚本目录
cd livepulse-connector-google-ads-open/docs/sql

# 执行建表脚本
psql -U google_ads_user -d livepulse_google_ads -f google_ads_schema.sql
```

### 4. 配置 Nacos

#### 4.1 导入配置

在 Nacos 控制台创建以下配置：

**Data ID**: `postgresql-config.yml`
**Group**: `DEFAULT_GROUP`

```yaml
spring:
  datasource:
    driver-class-name: org.postgresql.Driver
    url: jdbc:postgresql://postgresql-host:5432/livepulse_google_ads
    username: google_ads_user
    password: your_password
    hikari:
      minimum-idle: 5
      maximum-pool-size: 20
      idle-timeout: 300000
      connection-timeout: 30000
      max-lifetime: 1200000

mybatis-plus:
  mapper-locations: classpath*:/mapper/**/*.xml
  type-aliases-package: com.cs.entity
  configuration:
    map-underscore-to-camel-case: true
    cache-enabled: true
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
  global-config:
    db-config:
      id-type: auto
      logic-delete-field: deleted
      logic-delete-value: 1
      logic-not-delete-value: 0
```

**Data ID**: `kafka-config.yml`
**Group**: `DEFAULT_GROUP`

```yaml
spring:
  kafka:
    bootstrap-servers: kafka-host:9092
    consumer:
      group-id: google-ads-connector-group
      auto-offset-reset: latest
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
```

### 5. 构建项目

```bash
# 进入项目根目录
cd livepluse-connectorx

# 编译打包
mvn clean package -DskipTests

# 查看生成的 JAR 文件
ls -lh livepulse-connector-google-ads-open/target/livepulse-connector-google-ads-open.jar
```

### 6. 配置应用

创建 `application-prod.yml` 文件：

```yaml
server:
  port: 23009

spring:
  application:
    name: livepulse-connector-google-ads-open

# Google Ads 定时任务配置
google-ads:
  scheduled:
    enabled: true
    customer-match-sync:
      cron: "0 0 */6 * * ?"  # 每 6 小时执行一次
```

### 7. 启动应用

#### 方式一：直接启动

```bash
java -jar livepulse-connector-google-ads-open/target/livepulse-connector-google-ads-open.jar \
  --spring.profiles.active=prod \
  --NACOS_SERVER_ADDR=nacos-host:8848
```

#### 方式二：使用 Docker 启动

```bash
# 构建镜像
cd livepulse-connector-google-ads-open
docker build -t livepulse-connector-google-ads:1.0 .

# 启动容器
docker-compose up -d

# 查看日志
docker-compose logs -f
```

### 8. 配置 Nginx（可选）

```nginx
upstream google-ads-connector {
    server localhost:23009;
}

server {
    listen 80;
    server_name google-ads.yourdomain.com;

    location / {
        proxy_pass http://google-ads-connector;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## Google Ads API 配置

### 1. 创建 Google Ads Connected App

1. 访问 [Google Ads API Center](https://ads.google.com/apicenter)
2. 创建新的开发者令牌（Developer Token）
3. 创建 OAuth 2.0 客户端 ID 和密钥
4. 配置回调 URL

### 2. 获取 OAuth 2.0 刷新令牌

使用 OAuth 2.0 Playground 获取刷新令牌：

1. 访问 [OAuth 2.0 Playground](https://developers.google.com/oauthplayground/)
2. 配置 OAuth 2.0 凭据
3. 授权并获取刷新令牌

### 3. 配置 Google Ads 连接

通过 API 添加 Google Ads 连接配置：

```bash
curl -X POST http://localhost:23009/google-ads/store \
  -H "Content-Type: application/json" \
  -d '{
    "developerToken": "your_developer_token",
    "clientCustomerId": "123-456-7890",
    "clientId": "your_client_id",
    "clientSecret": "your_client_secret",
    "refreshToken": "your_refresh_token",
    "enabled": true
  }'
```

## 故障排查

### 1. 检查日志

```bash
# Docker 日志
docker-compose logs -f

# 应用日志
tail -f logs/livepulse-connector-google-ads-open.log
```

### 2. 常见问题

#### 问题 1：无法连接到 PostgreSQL

检查数据库连接配置：

```bash
psql -U google_ads_user -d livepulse_google_ads -h postgresql-host
```

#### 问题 2：无法注册到 Nacos

检查 Nacos 地址配置：

```bash
curl http://nacos-host:8848/nacos/v1/ns/instance/list?serviceName=livepulse-connector-google-ads-open
```

#### 问题 3：Google Ads API 调用失败

检查 API 凭据配置：
- 确认开发者令牌有效
- 确认 OAuth 2.0 凭据正确
- 确认客户 ID 格式正确

## 安全建议

1. **敏感数据加密**：对 OAuth 2.0 凭据进行加密存储
2. **访问控制**：配置 Nginx IP 白名单
3. **HTTPS 配置**：生产环境使用 SSL 证书
4. **日志脱敏**：避免在日志中输出敏感信息
